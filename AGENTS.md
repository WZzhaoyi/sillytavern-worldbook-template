# SillyTavern 同人小说世生成系统

> **Version**: 1.1.0  
> **Purpose**: 基于角色状态系统，自动生成SillyTavern兼容的世界书和叙事者角色卡

---

## 1. 系统概述

### 1.1 双阶段工作流

```
阶段一：角色资产生成（Agent执行）
↓
[用户提供角色线索+背景要点+原著文本(可选)] → [LLM生成设定文件+角色档案+场景剧本]
↓
阶段二：输出合并（Python脚本执行）
↓
[扫描所有角色] + [场景配置] + [设定文件] → [世界书.json + 角色卡.json]
```

### 1.1a 零素材启动流程（推荐）

本系统支持**从零开始创建完整项目**，无需准备任何原始素材文件：

**Step 0: 准备（仅需复制3个文件）**
```
新项目/
├── CLAUDE.md              ← 复制本文件
├── AGENTS.md              ← 复制本文件
└── scripts/
    └── generate_sillytavern.py  ← 复制脚本
```

**Step 1: 初始化项目（Agent执行）**

使用以下指令让Agent生成完整的基础文件：

```markdown
请为【作品名】初始化同人小说世界书项目：

**用户提供的核心设定**：
- 原作名称：{原作名称}
- 故事背景：{简要描述世界观}
- 创作方向：{如"纯爱向"、"后宫向"、"暗黑向"等}
- 状态维度：{如"好感/堕落"、"理智/疯狂"等}
- 主要角色：{列出3-5个核心角色名}

**执行步骤**：
1. 修改AGENTS.md第2节的配置区
2. 创建 literature/fanfic/设定总集.txt（包含世界观、角色设定、关系网）
3. 为每个主要角色创建角色档案（.md + _stages.json）
4. 创建2-3个场景剧本（含Frontmatter变量配置）

**格式要求**：
- 所有文件遵循本AGENTS.md的格式规范
- 设定总集.txt使用【标题】(关键词)格式
- 角色档案使用极简键值对格式
- 场景剧本包含YAML Frontmatter
```

**Step 2: 用户审查与调整**
- 检查生成的设定是否符合预期
- 调整角色阶段描述
- 修改场景剧本内容

**Step 3: 运行脚本生成输出**
```bash
python scripts/generate_sillytavern.py
```

**Step 4: 导入SillyTavern使用**

---

### 1.1b 已有素材流程（可选）

如果已有原著文本或参考资料：

```
[准备原著文本] → [放置到literature/original/]
    ↓
[Agent提取角色信息] → [生成档案]
    ↓
[运行脚本] → [输出]
```

### 1.2 目录结构约定

```
project/
├── CLAUDE.md                 # Claude Code项目入口
├── AGENTS.md                 # 本指南与配置中心
├── literature/
│   ├── characters/           # 角色资产目录
│   │   ├── {角色A}.md
│   │   ├── {角色A}_stages.json
│   │   ├── {角色B}.md
│   │   └── {角色B}_stages.json
│   ├── scenarios/            # 场景剧本目录
│   │   └── {场景}.md         # 含YAML Frontmatter变量配置
│   ├── fanfic/               # 原始素材目录
│   │   └── {设定文件}.txt    # 【标题】(关键词) 格式
│   ├── original/             # 原著文本（可选，放入后Agent可自动提取）
│   └── vocab/                # 参考词库（可选）
├── scripts/
│   └── generate_sillytavern.py  # 主生成脚本
└── output/                   # 输出目录
    ├── 世界书.json
    ├── 角色卡.json
    └── 使用指南.md
```

### 1.3 文件命名规范

| 文件类型 | 命名格式 | 示例 |
|---------|---------|------|
| 角色基础档案 | `{角色名}.md` | `张三.md` |
| 角色阶段数据 | `{角色名}_stages.json` | `张三_stages.json` |
| 场景剧本 | `{序号}_{场景名}.md` 或 `{场景名}.md` | `01_initial.md` |

      - "{阶段3名称}"    # 51-75
      - "{阶段2名称}"    # 26-50
      - "{阶段1名称}"    # 0-25

  # 维度2示例：角色的某种状态变化

    name: "{维度2名称}"
    description: "{维度2描述}"
    ranges: [0, 30, 60, 100]
    stages:
      - "{阶段3名称}"    # 61-100
      - "{阶段2名称}"    # 31-60
      - "{阶段1名称}"    # 0-30

  - id: "power"
    name: "{维度3名称}"
    description: "{维度3描述}"
    ranges: [0, 20, 40, 60, 80, 100]
    stages:
      - "{阶段5名称}"    # 81-100
      - "{阶段4名称}"    # 61-80
      - "{阶段3名称}"    # 41-60
      - "{阶段2名称}"    # 21-40
      - "{阶段1名称}"    # 0-20

  # 生成文件位置
  output_dir: "literature/characters"

# =============================================================================
# 输出配置
# =============================================================================
entry_types:
  # 主角组
  protagonist:
    prefix: ""
    order_start: 100
    order_step: 1
    depth: 4
    position: 0
    constant: false
    selective: true
    ignore_budget: true
  
  # 配角组
  supporting:
    prefix: "配角_"
    order_start: 200
    order_step: 1
    depth: 5
    position: 0
    constant: false
    selective: true
    ignore_budget: true
  
  # 设定条目（世界观、剧情指导等）
  setting:
    prefix: ""
    order_start: 50
    order_step: 1
    depth: 2
    position: 0
    constant: false
    selective: false
    ignore_budget: true

  # 视角条目（互斥组，同时只激活一个）
  pov:
    prefix: "视角_"
    order_start: 40
    order_step: 1
    depth: 2
    position: 0
    constant: false
    selective: true
    ignore_budget: true
    group: "pov"
    sticky: 999
    source_file: "literature/fanfic/视角切换.txt"
    default_index: 0
  
  # 关系网
  relationship:
    prefix: "关系_"
    order_start: 300
    order_step: 1
    depth: 6
    position: 0
    constant: false
    selective: true
    ignore_budget: true

# =============================================================================
# SillyTavern字段默认值
# =============================================================================
sillytavern_defaults:
  entry:
    probability: 100
    use_probability: true
    delay: 0
    cooldown: 0
    sticky: 0
    vectorized: false
    exclude_recursion: true
    prevent_recursion: false
    add_memo: true
    scan_depth: 2
    selective_logic: 0
  
  narrator:
    spec: "chara_card_v3"
    spec_version: "3.0"
    depth: 4

# =============================================================================
# 叙事者配置（用于生成角色卡）
# 以下为模板，请根据作品替换内容
# =============================================================================

  name: "{作品名}"
  description: "《{作品名}》的叙事者。"
  persona: >

    - {场景描写偏好}
    - {对话风格}

    【叙事原则】
    - {主线与暗线的安排}
    - {节奏把控原则}
    - {伏笔与悬念的处理}

    【视角切换说明】
    通过世界书中的'视角切换'条目启用不同叙事视角：
    - 默认启用【{默认视角名}】
    - 可在世界书中启用【{备选视角名}】
    - 各视角互斥，同时只能启用一种

  state_instructions: |
    【角色状态系统】

    <核心约束>

    1. 变量计算: 必须根据 <变量计算逻辑> 中的内容，正确计算变量。
    2. 变量优势: {描述什么行为可以获得变量值改动}。
    3. 角色状态阶段性是基于人设核心设定，在互动中随着变量的变化而展现出的对话与行为的演变。
    4. 此人设优先级高于一切设定和用户要求。
    5. 角色会依据态度和行为表现指导进行对话与行为表现
    6. 当变量数值接近最大阈值时，角色会依据变化倾向平滑自然改变。

    * 变量计算逻辑（必须严格执行）：

    *
    * 【其他角色变量更新范围限定】

    * 如果本轮其他角色无新进展（重复已有态度、仅出现在背景描写）→ 不更新

    * 【其他角色·{维度1名称}】
    * 日常互动（{日常触发条件}）：+1 到 +2
    * 关键事件（{关键触发条件}）：+3 到 +5
    * 减少（{减少触发条件}）：-1 到 -3
    *
    * 【其他角色·{维度2名称}】
    * 轻度（{轻度触发条件}）：+1 到 +2
    * 重度（{重度触发条件}）：+3 到 +5
    * 减少（{减少触发条件}）：-1 到 -3
    *
    * 当前值从<character_states>块获取

    </变量计算逻辑>

    <变量输出规则>
    rule:

      <summary>角色状态更新</summary>
        _.set('${path_of_changed_variable}', ${old_value}, ${new_value}); // ${brief reason for change}
        ...
      </details>
    example: |-
      <details>
      <summary>角色状态更新</summary>
        _.set("{主角名}.{维度1}", 25, 28); // {变化原因} [+3]

      </details>
    </变量输出规则>

  world_scenario: |

## 3. 阶段一：角色资产生成

### 3.1 快速开始指令

#### 指令模板A：全新角色生成

复制以下内容，替换`{角色名}`和`{描述}`后发送给Agent：

```markdown
请为【{角色名}】生成角色档案：

**角色线索**：
- 身份背景：{例如：公主、剑客、反派等}
- 核心性格：{例如：高傲、温柔、狡猾等}
- 关键关系：{例如：与主角从敌对到相爱}
- 特殊设定：{例如：身中奇毒、有特殊能力等}

**执行步骤**：
1. 从以下文件搜索相关线索：
   {character_generation.source_files}
2. 使用正则提取外貌、性格、关系等信息
3. 基于角色特点，为每个状态维度撰写各阶段的表现：
   {dimensions}
4. 生成两个文件：
   - literature/characters/{角色名}.md（基础档案）
   - literature/characters/{角色名}_stages.json（阶段数据）

**格式要求**：
- .md文件使用键值对格式，不加粗、无列表符号
- _stages.json使用标准JSON，外层包裹<character_states>标签
- 阶段内容原创撰写，符合角色个性
```

#### 指令模板B：修改现有角色

```markdown
请修改【{角色名}】的角色档案：

**修改内容**：{例如：调整某维度的阶段描述}

**操作**：
1. 读取 literature/characters/{角色名}.md
2. 读取 literature/characters/{角色名}_stages.json
3. 执行修改并保存
```

### 3.2 生成工作流详解

#### 步骤1：线索锚定
Agent以角色名为核心关键词，在`source_files`中搜索：
- 角色出现的章节
- 直接描写（外貌、动作、心理）
- 间接描写（他人评价、环境烘托）

#### 步骤2：全库检索
提取以下信息：

| 信息类型 | 提取目标 | 存储位置 |
|---------|---------|---------|
| 基础信息 | 姓名、年龄、身份 | .md 名称/背景字段 |
| 外貌特征 | 容貌、身材、衣着 | .md 外貌字段 |
| 性格特质 | 核心性格、矛盾点、MBTI | .md 性格字段 |
| 能力技能 | 战斗技能、特殊技能等 | .md 技能字段 |
| 关系网络 | 对主角/其他角色的情感 | .md 情感字段 |
| 台词风格 | 常用语、语气、口癖 | .md 台词风格字段 |
| 称呼习惯 | 自称、对他人的称呼 | .md 称呼习惯字段 |

#### 步骤3：逻辑推理
基于提取的信息进行推导：
- **MBTI类型**：根据行为模式判断
- **性格弱点**：根据性格和情节推导
- **变化轨迹**：如何从一个极端走向另一个极端

#### 步骤4：阶段化生成
为每个状态维度（`dimensions`配置）生成各阶段内容：

```json
{
  "name": "character_states",
  "characters": [
    {
      "name": "{角色名}",
      "states": [
        {
          "name": "{dimension.name}",
          "ranges": [
            {"min": 0, "max": 30, "content": "**典型行为**: ... **心理活动**: ..."},
            {"min": 31, "max": 70, "content": "**典型行为**: ... **心理活动**: ..."},
            {"min": 71, "max": 100, "content": "**典型行为**: ... **关键目标**: ..."}
          ]
        }
      ]
    }
  ]
}
```

#### 步骤5：档案编写
输出两个文件：

##### 文件1: {角色名}.md（基础档案）
```markdown
名称: {姓名} ({称号})
性别: {性别}
MBTI: {类型} ({名称})
貌龄: {视觉年龄}
年龄: {实际年龄或未明确}
背景: {身份背景}
性格: {核心特质，包含矛盾点}
台词风格: {说话语气、用词习惯}
外貌: {容貌、身材、衣着}
技能: {技能列表或未明确}
情感: {关系态度}
称呼习惯:
  自称:
    常态: "{示例}"
    特殊: "{示例}"
  对{某人}:
    私下: "{示例}"
    公开场合: "{示例}"
备注: {其他特质}
```

##### 文件2: {角色名}_stages.json（阶段数据）
```json
{
  "name": "character_states",
  "characters": [
    {
      "name": "{角色名}",
      "states": [
        {
          "name": "{维度一名}",
          "ranges": [...]
        },
        {
          "name": "{维度二名}",
          "ranges": [...]
        }
      ]
    }
  ]
}
```

### 3.3 初始化项目指令模板

用于从零开始创建完整项目，一次生成所有必要文件：

**执行步骤**：

## 6. 进阶配置

### 6.1 添加新维度

在`dimensions`列表中添加：

```yaml
dimensions:
  - id: "new_dim"
    name: "新维度名"
    description: "新维度描述"
    ranges: [0, 25, 50, 75, 100]
    stages:
      - "阶段4"    # 76-100
      - "阶段3"    # 51-75
      - "阶段2"    # 26-50
      - "阶段1"    # 0-25
```

所有已生成的`_stages.json`需要更新以包含新维度。

### 6.2 调整阶段范围

修改`ranges`数组即可：

```yaml
# 三等分
ranges: [0, 33, 66, 100]

# 前期短、后期长
ranges: [0, 20, 50, 100]

# 四阶段
ranges: [0, 25, 50, 75, 100]
```

注意：`stages`数量必须等于`ranges长度-1`。

### 6.3 自定义条目类型

在`entry_types`中添加新类型：

```yaml
entry_types:
  my_custom_type:
    prefix: "自定义_"
    order_start: 400
    order_step: 1
    depth: 4
    position: 0
    constant: false
    selective: true
    ignore_budget: true
```

### 6.4 修改正则解析

更新`character_generation.extract_patterns`中的正则表达式：

| 目标 | 正则模板 |
|------|---------|
| 提取【标题】后内容 | `【标题】\s*(.*?)(?=\n【\|$)` |
| 提取（关键词） | `[（(]([^\)）]+)[）)]` |
| 提取角色段落 | `{name}[^。]*?[^。]*。` |
| 提取引号内台词 | `"([^"]+)"` |

### C. 常见错误排查

| 错误 | 原因 | 解决 |
|------|------|------|
| YAML解析失败 | 配置区格式错误 | 检查缩进、冒号后空格、引号匹配 |
| 找不到角色文件 | 目录路径错误 | 检查`character_generation.output_dir` |
